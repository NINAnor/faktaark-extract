# faktaark

A CLI tool that fetches Norwegian nature-type (naturtype) fact sheets from
Miljødirektoratet's ArcGIS REST API and uses an AI model to extract structured
ecological risk assessments.

## Pipeline

```
input file (.csv / .parquet / .json / …)
        │
        ▼  faktaark fetch <INPUT_FILE> <COLUMN>
output/naturtyper.parquet           (raw ArcGIS data, one row per locality)
        │
        ▼  faktaark extract
output/naturtyper_analyse.parquet   (AI-extracted fields, joinable on naturtype_id)
```

## Requirements

- Python ≥ 3.10
- [uv](https://docs.astral.sh/uv/getting-started/installation/)

## Installation

```bash
uv sync
```

## Configuration

The `extract` command calls an LLM. Copy `.env.example` to `.env` and set the
API key for your chosen provider:

```bash
cp .env.example .env
```

| Provider | Environment variable | Model string prefix |
|---|---|---|
| Google Gemini (default) | `GOOGLE_API_KEY` | `google-gla:…` |
| OpenAI | `OPENAI_API_KEY` | `openai:…` |
| Anthropic Claude | `ANTHROPIC_API_KEY` | `anthropic:…` |
| Groq | `GROQ_API_KEY` | `groq:…` |
| Mistral | `MISTRAL_API_KEY` | `mistral:…` |

> **Never commit your `.env` file.**

## Usage

### `fetch`

Reads a tabular input file, extracts the column containing `naturtypeId` values,
and fetches the full fact-sheet records from the Miljødirektoratet ArcGIS REST API.

```bash
faktaark fetch <INPUT_FILE> <COLUMN> [OPTIONS]
```

| Argument / Option | Default | Description |
|---|---|---|
| `INPUT_FILE` | *(required)* | Input file — `.csv`, `.parquet`, `.json`, `.ndjson`, `.arrow`, `.feather`, or `.orc` |
| `COLUMN` | *(required)* | Name of the column whose values are used as `naturtypeId` |
| `--batch-size` | `100` | Number of IDs per API request |
| `--output` | `output/` | Directory where the output parquet is written |
| `-v` / `--verbose` | `False` | Enable debug logging |

**Example** — fetch all localities listed in `data.csv`:

```bash
uv run main.py fetch data.csv identifikasjon_lokalId
```

Produces `output/naturtyper.parquet`.

---

### `extract`

> **Experimental.** The AI extraction pipeline is experimental. Its output is
> generated by a language model and may contain errors, omissions, or
> hallucinations. Do not rely on the results without manual review.

Reads the parquet produced by `fetch`, sends each row's Norwegian text
descriptions to an AI model, and writes the structured ecological assessments
to a separate parquet file joinable on `naturtype_id`.

The primary extracted field is `gjengroing_risiko`, which indicates whether
the locality is at risk of or affected by reforestation (gjengroing).

```bash
faktaark extract [OPTIONS]
```

| Option | Default | Description |
|---|---|---|
| `--input` | `output/naturtyper.parquet` | Input parquet file produced by `fetch` |
| `--output` | `output/naturtyper_analyse.parquet` | Output parquet with AI-extracted fields |
| `--model` | `google-gla:gemini-3-flash-preview` | PydanticAI model string (see [Configuration](#configuration)) |
| `--concurrency` | `10` | Maximum number of parallel API calls |
| `--rpm` | `5.0` | Maximum requests per minute (rate limit) |
| `--max-retries` | `12` | Retries per row on rate-limit (429) errors |
| `--limit` | *(none)* | Process only the first N rows — useful for testing |
| `-v` / `--verbose` | `False` | Enable debug logging |

**Example** — extract using Google Gemini:

```bash
uv run main.py extract --model google-gla:gemini-2.0-flash
```

**Example** — test on the first 10 rows using OpenAI:

```bash
uv run main.py extract --model openai:gpt-4o-mini --limit 10
```

Produces `output/naturtyper_analyse.parquet`.

## Output schema

### `output/naturtyper.parquet`

One row per nature locality. Key columns returned by the ArcGIS API:

| Column | Description |
|---|---|
| `naturtype_id` | Unique locality identifier (e.g. `BN00007512`) — join key |
| `naturtype` | Nature type name |
| `verdi` | Conservation value assessment |
| `hevdstatus` | Management condition code (see [Domain glossary](#domain-glossary)) |
| `omraadenavn` | Locality name |
| `innledning` | Introduction / general description |
| `artsmangfold` | Species richness description |
| `fremmede_arter` | Invasive / alien species description |
| `raad_om_skjoetsel_og_hensyn` | Management and conservation advice |

### `output/naturtyper_analyse.parquet`

One row per locality. All columns are `null` for rows where extraction failed.

| Column | Type | Description |
|---|---|---|
| `naturtype_id` | string | Join key — matches `naturtype_id` in `naturtyper.parquet` |
| `gjengroing_risiko` | bool | **Primary flag** — is the area at risk of or affected by reforestation? |
| `gjengroing_grad` | string | Current degree of overgrowth: `ingen` / `lav` / `moderat` / `høy` |
| `gjengroing_beskrivelse` | string\|null | Narrative description of the overgrowth situation |
| `skjoetsel_anbefalt` | bool | Is active management recommended to preserve nature values? |
| `skjoetsel_tiltak` | list[string] | Specific recommended management actions |
| `fremmede_arter_liste` | list[string] | Invasive or alien species named in the text |
| `paavirkningsfaktorer` | list[string] | Identified threat and impact factors |
| `tilstand` | string | Overall ecological condition: `god` / `moderat` / `dårlig` / `ukjent` |
| `noekkelarter` | list[string] | Key species or important nature values mentioned |

## Domain glossary

| Term | Meaning |
|---|---|
| **naturtype** | Nature type — a defined category of habitat or ecosystem used in Norwegian biodiversity mapping |
| **faktaark** | Fact sheet — a locality-level descriptive record produced during nature-type surveys |
| **gjengroing** | Reforestation / overgrowth — the process by which open or semi-open habitats are encroached upon by shrubs and trees, reducing biodiversity |
| **hevdstatus** | Management condition — a code describing whether a locality is actively managed: 1 = god hevd (good), 2 = svak hevd (weak), 3 = ingen hevd / restaureringstilstand (none / restoration needed), 4 = gjengrodd (overgrown), 5 = ikke vurdert (not assessed) |
| **skjøtsel** | Nature management / stewardship — active interventions such as grazing, mowing, or clearing to maintain habitat quality |
| **hevd** | The condition of a semi-natural habitat as a result of traditional land use; lost when management ceases |
